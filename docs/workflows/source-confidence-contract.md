# Source Confidence Contract

Purpose: define source capabilities, confidence defaults, freshness expectations, supported claims, forbidden claims, and agent-specific source usage.

## Confidence Levels

- `high`: direct connected/uploaded source, recent, clear provenance, matches claim.
- `medium`: credible source but partial, noisy, or limited in scope.
- `low`: public, inferred, stale, incomplete, or only directional.
- `unknown`: source unavailable or provenance unclear.

Confidence must be lowered for stale data, missing provenance, contradictory evidence, or source mismatch.

## Brief Claim Rules

Today's Brief is user-facing and must obey source capability rules for every dynamic sentence.

- Stream counts require Spotify for Artists, distributor analytics, DSP analytics export, smart-link/provider data that reports streams, or another connected/uploaded source with the count and time window.
- Geography claims require a source that exposes geography for the relevant signal. Comments mentioning a city are not the same as listener geography, smart-link geography, ticketing geography, or streaming geography.
- Public comments can support phrasing like "Lagos comments repeated the lyric"; they cannot support "Lagos demand is confirmed" without stronger evidence.
- Regional growth percentages require previous and current comparable windows from the same source.
- "Generated by AI Manager 08:30 AM" must use the run or brief snapshot timestamp.
- Recent Movement items must link to operating events, source snapshots, agent notes, task results, checkpoint updates, reviews, or memory entries.
- Saved Music state can prevent unnecessary refetches, but any claim-bearing Music field must still trace to user input, source snapshot, evidence, provider confirmation, memory, or explicit limitation.

## Agent-Specific Use

The same source can support different conclusions by agent:

- Marketing uses social/content evidence for angle, creator, campaign, and audience hypotheses.
- Touring uses geography, city comments, streaming locations, and live history for city validation.
- Finance/Rights uses budget, royalties, splits, payout, metadata, and ownership documents; social attention is weak finance evidence.
- Sync & Deals uses rights clarity, clean assets, pitch materials, catalog metadata, and audience proof.
- Manager uses all sources for synthesis but must preserve each source limitation.

## Forbidden Claim Rules

Do not claim:

- private Spotify saves or source-of-stream without private data
- revenue/payout certainty without royalty statements
- rights/ownership certainty without documents and human/legal review
- conversion from social attention alone
- causation from correlation
- live-market readiness from comments alone
- campaign ROI from views alone

## Contradictions

If sources conflict:

- lower confidence
- surface the conflict
- create review or evidence request
- do not hide the weaker source
- do not let one agent's report override Manager synthesis

## Acceptance Checks

- Every evidence item has source, freshness, confidence, provenance, limitation, and raw reference where available.
- Every agent report states the lens used.
- Every decision package names missing evidence when confidence is limited.
